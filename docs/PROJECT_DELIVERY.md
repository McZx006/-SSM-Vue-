# 个人记账看板交付说明

## 项目定位

这是一个面向大学生日常记账场景的小型个人记账看板，采用 SSM 后端、Vue 3 前端、MySQL 数据库和 ECharts 图表。项目重点是把“记录账单”变成“可视化理解消费”，适合作为课程设计、小型毕设原型或继续改造成微信小程序的后端基础。

## 已完成功能

- 登录注册：支持账号注册、登录、登录态校验、退出登录和登录失败提示。
- 仪表板：首页展示本月收入、本月支出、本月结余、预算使用率、预算提醒、支出分类饼图、月度收支柱状图和最近账单。
- 快捷记账：登录后所有页面都有“记一笔”浮动按钮，支持早餐、通勤、教材、兼职等常用模板一键填充，也支持手动选择类型、金额、分类、日期和备注。
- 账单管理：支持新增、编辑、删除、分页查询、按类型/分类/日期范围/关键词筛选，以及本月、收入、支出快捷筛选。
- 分类管理：支持系统分类展示、用户自定义分类新增、编辑、删除和颜色管理。
- 统计分析：支持分类占比饼图、分类排行柱状图、月度趋势折线图、最高分类、年度支出峰值、全年结余洞察和智能消费建议。
- 系统设置：支持个人资料、预算、密码修改、数据导出、紧凑模式、深色模式等使用偏好。
- Excel 导出：支持按时间范围导出 `.xlsx` 账单表格。
- 移动端体验：小屏幕隐藏侧边栏，使用底部 Tab 导航和浮动记账按钮，更接近微信小程序的使用方式。
- 小程序原型：提供 `miniapp-prototype` 微信小程序原生页面骨架，包含登录、注册、快捷记账、首页、账单、分析、我的页面；首页、账单、记账和统计页面已接入当前 SSM 后端核心接口。

## 验收清单

- 可以运行 `powershell -ExecutionPolicy Bypass -File scripts/verify-project.ps1` 完成小程序 JS 语法、前端构建、后端打包和本地入口探测。
- Tomcat 启动并部署后，可以运行 `powershell -ExecutionPolicy Bypass -File scripts/api-smoke-test.ps1` 验证登录、分类、账单和统计接口。
- 后端地址 `http://localhost:8080/account-dashboard/api/auth/check` 未登录时返回 401 JSON，而不是乱码页面。
- 前端地址 `http://localhost:3000/` 能打开登录页。
- 使用测试账号 `testuser / 123456` 可以登录。
- 登录后仪表板有饼图和柱状图，最近账单列表能显示数据。
- 任意登录后页面右下角有“记一笔”按钮，可用常用模板快速填充账单，新增后当前页面数据会自动刷新。
- 账单管理可以新增、修改、删除账单，并能使用快捷筛选。
- 统计分析有饼图、排行柱状图、趋势折线图和洞察卡片。
- 首页和统计页能根据预算、分类排行和年度趋势生成智能建议。
- 系统设置可以维护分类、修改预算并导出 Excel。
- 系统设置可以开启深色模式，页面主题会立即切换并保存偏好。
- `miniapp-prototype` 可以导入微信开发者工具，登录、注册、首页摘要、快捷记账、账单列表和统计分析页面具备基础校验与接口调用逻辑。

## 演示建议

1. 先登录测试账号，展示首页预算状态和图表。
2. 在账单管理新增一条支出，返回首页观察数字和图表变化。
3. 打开统计分析，切换月份和年份，展示分类占比与月度趋势。
4. 打开系统设置，新增一个自定义分类并设置颜色。
5. 在数据导出区域选择本月，导出 Excel 文件。

## 数据库脚本

- `src/main/resources/sql/schema.sql`：完整建库、建表、系统分类和测试账单数据。
- `src/main/resources/sql/demo_statistics_2026.sql`：用于补充统计图表演示数据。
- `src/main/resources/sql/localize_demo_data.sql`：旧数据库中文化补丁，把英文分类和英文备注更新为中文。

## 可继续扩展

- 微信小程序端：复用后端 RESTful 接口，使用小程序原生组件或 uni-app 实现移动端。
- 预算提醒：增加多分类预算、支出接近预算时弹窗提醒。
- AI 分析：当前已实现本地规则版消费建议，后续可替换为真正的大模型接口。
- OCR 识别：上传票据图片后自动提取金额、商户、日期和分类。
- 语音记账：支持“今天午饭 18 元”这类自然语言录入。
